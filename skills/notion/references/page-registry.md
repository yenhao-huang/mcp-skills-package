# Common Notion Page Registry

Use this file as the local map of pages, databases, and data sources the user
commonly works with. Keep entries short and operational.

## Registry Rules

- Add an entry when the user calls a page common, asks to remember it, or uses it
  repeatedly in a workflow.
- Prefer canonical Notion URLs when available. Store raw IDs only when no URL is
  available.
- Keep aliases stable and human-friendly. Include Chinese aliases when the user
  uses them.
- For databases, record both the database URL/ID and the relevant
  `collection://...` data source ID after fetch reveals it.
- Do not store secrets, private tokens, or unrelated page content here.
