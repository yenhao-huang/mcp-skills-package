# Issue Body Templates

Use repository-provided templates first. Use these only when the repository has
no suitable template.

## Bug Report

```markdown
## Description

Describe the observed problem and its impact.

## Steps To Reproduce

1. First step
2. Second step
3. Observe the failure

## Expected Behavior

Describe the expected outcome.

## Actual Behavior

Describe the actual outcome, including sanitized logs when useful.

## Environment

List relevant version and platform details.
```

## Feature Request

```markdown
## Summary

Describe the requested capability.

## Motivation

Explain the user problem and why it matters.

## Proposed Behavior

Describe the desired behavior without prescribing unnecessary implementation.

## Acceptance Criteria

- [ ] Observable outcome one
- [ ] Observable outcome two
```

## Task

```markdown
## Goal

State the outcome to deliver.

## Scope

List included work and explicit exclusions.

## Acceptance Criteria

- [ ] Completion condition one
- [ ] Completion condition two

## Dependencies

List related or blocking issues when known.
```
